C — Preview + inline edits

Show a preview card of what recipients see. Every field has a small pencil to edit in place via a tiny sheet. Feels the most “dead simple” but more component work.

My recommendation: A

  • Simplest to build with existing modal patterns (CodeEntryModal-style)
  • One scroll, no navigation state to manage
  • Matches “dead simple” — required fields first, optional collapsed
  • Easy to A/B against future variants if needed

When I’d pick B or C instead

  • B if you want to measure step-level dropoff, or if the sheet feels too dense in practice
  • C if you want the most polished feel and have time for the preview-card component
